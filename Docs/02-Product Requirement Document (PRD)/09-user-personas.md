# Second-Hand Marketplace - User Personas

## Persona P1 - Rafi (University Student Seller)
| Attribute | Details |
|---|---|
| Demographics | 21, third-year CS student |
| Goals | Sell old textbooks and electronics before semester ends |
| Motivations | Make money from unused items, declutter |
| Pain Points | Listings buried in Facebook groups, no tracking of buyer interest |
| Technology Usage | Laptop and mobile web |
| Typical Workflow | Take photo -> create listing -> receive message -> mark as sold |

## Persona P2 - Nadia (Young Professional Buyer)
| Attribute | Details |
|---|---|
| Demographics | 26, recently relocated professional |
| Goals | Find affordable second-hand furniture and electronics |
| Motivations | Save money, find quality items quickly |
| Pain Points | Cannot filter by price or condition; wastes time scrolling |
| Technology Usage | Mobile-first, browses during commute |
| Typical Workflow | Search keyword -> filter category/price -> view listing -> message seller |

## Persona P3 - Karim (Frequent Reseller)
| Attribute | Details |
|---|---|
| Demographics | 30, part-time reseller with multiple active listings |
| Goals | Manage 10–20 listings at a time and track sold status |
| Motivations | Side income, fast turnover |
| Pain Points | No dashboard, loses track of which items are sold |
| Technology Usage | Desktop-first, keyboard-heavy |
| Typical Workflow | Post listing -> monitor messages -> mark sold -> post next item |

## Persona P4 - Sadia (Graduate Student Buyer)
| Attribute | Details |
|---|---|
| Demographics | 24, graduate student on tight budget |
| Goals | Find study equipment and lab supplies cheaply |
| Motivations | Affordability, specific item needs |
| Pain Points | No condition information, cannot trust vague descriptions |
| Technology Usage | Desktop browser |
| Typical Workflow | Search keyword -> filter condition -> view photos -> save to favorites -> message seller |
